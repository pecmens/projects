# 静态状态发布系统 (SSPS)

一个轻量级的博客生成和管理系统，允许用户在本地开发环境中创建、管理和发布静态博客内容。

## 概述

静态状态发布系统(SSPS)是为希望使用Next.js + React作为前端在本地开发环境中创建博客的开发人员而设计的，同时使用各种后端语言实现内容管理功能。该系统最终生成可以部署到GitHub的静态页面。

## 核心功能

### 本地开发环境
- 轻量级本地环境，依赖项最少
- 在本地开发过程中同时显示前端和后端
- 本地环境模拟在线服务器行为，无环境差异

### 多语言后端支持
- 默认后端：Python(Django)、Go(Gin)和C#(ASP.NET Core)
- 清晰的API接口，支持与其他后端语言集成
- 用于内容管理(文章、页面、分类等)的后端管理系统

### 静态博客生成
- 通过后端管理系统自动生成静态HTML页面
- 支持Markdown格式的内容创建
- 图片和附件资源管理
- 完整的静态文件结构，可直接部署到GitHub Pages

### GitHub部署
- 一键上传到GitHub的命令行工具
- 支持GitHub Pages自动部署
- 将生成的静态文件直接上传到GitHub仓库

## 技术要求

### 前端
- 基于Next.js 14+构建
- React 18+支持TypeScript
- 使用Tailwind CSS进行样式设计
- 实时预览功能

### 后端
- RESTful API设计
- 支持JWT认证
- 数据存储：SQLite(本地开发)，可扩展到其他数据库

### 本地环境
- 最少依赖(Node.js和可选的Python)
- Docker Compose支持轻松环境设置
- 无需额外的数据库或复杂服务
- 安装简单，无需复杂配置

## 产品愿景

SSPS致力于为用户提供一个简单、轻量级和多功能的博客发布平台，让他们专注于内容创作，而无需担心复杂的后端架构或部署过程。通过将本地开发环境与GitHub静态页面托管相结合，SSPS提供了最佳的博客体验，同时保持了静态网站的高性能和安全性。
